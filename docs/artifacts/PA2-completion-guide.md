# PA2 Completion Guide

Use this guide to finish all PA2 deliverables for the Freestyle Chess mobile website project. The main rule is: every claim should connect to evidence from the survey, interview clips, observation notes, or screenshots.

## Required Submission Files

| Requirement | Final File | Main Source Artifacts |
|---|---|---|
| R1 User Research | `Group06-PA2-UserResearch.pdf` | Survey CSV, interview clips, observation notes, personas, work models |
| R2 User Analysis | `Group06-PA2-UserAnalysis.pdf` | Brainstorm notes, affinity diagram, prioritized problems |
| R3 Proposal | `Group06-PA2-ProjectProposal.pdf` | Selected pain points, concept alternatives |
| R4 Use Case Specification | `Group06-PA2-UseCaseDocument.pdf` | Use case diagrams/flow models, use case specs |
| R5 Peer Review | `Group06-PA2-PeerReview.pdf` | Presentation feedback and group responses |
| R6 Weekly Report | `Group06-PA2-WeeklyReport.pdf` | Member task tracking |

## Member Workflow For Interview Clips

Each member reviews the clip they recorded or conducted. Fill `R1-interview-clip-review-template.md` for that participant, then send the completed notes to the group lead. Use participant codes only, such as `P01`, `P02`, and `P03`.

For each task, record the user goal, actual steps, hesitation, scrolling, accidental taps, grip changes, quote, result, and design implication. This follows LN04 Task Analysis: describe the task, decompose the steps, identify breakdowns/exceptions, and explain what the interface should improve.

## R1 Completion Checklist

- Confirm interview count, duration, date range, and recording method.
- Complete participant notes for every interview clip.
- Replace all `[ ]`, `[Fill in]`, and placeholder quotes in `06-PA2-UserResearch.md`.
- Add at least 3 personas based on survey/interview patterns.
- Complete Flow Model, Sequence Model, and Artifact Model.
- Add evidence references: screenshot names, clip timestamps, or observation notes.
- State research limitations honestly.

## R2 Completion Checklist

- Convert survey findings and interview observations into short sticky notes.
- Separate notes into problem notes and solution idea notes.
- Group notes into themes: navigation, homepage clarity, search/filter, video behavior, and visual cues.
- Create an affinity diagram in Miro, Figma, FigJam, or another visual tool.
- Export the diagram as an image and include it in the report.
- Prioritize 3-5 severe user problems using frequency, severity, and evidence strength.

## R3 Completion Checklist

- Select only a few tough problems from R2.
- Explain why these problems are more important than the others.
- Propose 2-3 distinct conceptual solutions for each selected problem.
- Compare concepts by user value, feasibility, risk, and connection to evidence.
- Choose the recommended concept direction for later prototype work.

## R4 Completion Checklist

- For each selected concept, draw a use case diagram or flow model.
- Identify actors, system/interface artifacts, and user-system interactions.
- Write use case specs with main flow, alternative flows, exceptions, and postconditions.
- Keep use cases user-goal oriented, not screen-by-screen UI descriptions only.

## R5 And R6 Completion Checklist

For peer review, document who commented, what they asked, how the group responded, and what changed afterward. For weekly report, list each member's task, output, completion status, and evidence link.
